Learning SQL Programming

Introductie

SQL programmeren leren

De wereld wordt gedreven door data en veel van deze data is opgeslagen in databases. Om deze informatie en inzichten te kunnen gebruiken moet de data worden opgevraagd. SQL is een veelgebruikte taal voor het opvragen en wijzigen van gegevens in databases.

DB Browser voor SQLite verkennen

Tijdens het werken met SQL is er een tool nodig waarin statements kunnen worden geschreven en die deze statements naar de database software kan sturen. Er zijn veel verschillende programma's hiervoor beschikbaar, waaronder SQLite.

In SQLite is de structuur van de database terug te zien in het "Database Structure" tabblad. De data in de tabel kan doorzocht worden in "Browse Data" overzicht en het schrijven en uitvoeren van SQL-statements met behulp van het tabblad "Execute SQL".

Om een statement uit te voeren kan de play button of de f5 shortcut worden gebruikt. Wanneer een statement wordt uitgevoerd, zal er een bericht worden teruggegeven met de status. Dit is handig voor troubleshooten.

Alle statements in de query box worden uitgevoerd. Om slechts één query uit te voeren moet de rest worden verwijderd of moet de query worden geselecteerd.

De SQL-log is een overzicht in SQLite die alle uitgevoerde opdrachten in een project bijhoudt. Als een statement moet worden verwijderd, zorgt het de SQL-log dat het niet voor altijd is verdwenen.

Wat is een database?

Een database is een verzameling van informatie die is georganiseerd in kolommen (velden) en rijen (records). Velden en records vormen een tabel, en één of meer tabellen vormen een database.

De lay-out en de manier waarop velden, tabellen en relaties zijn opgezet in een database, wordt een schema genoemd.

In tegenstelling tot spreadsheets kunnen databases complexe queries en relaties tussen tabellen aan. SQL wordt hierbij voor interactie met databases gebruikt, waardoor het gemakkelijker wordt om gegevens op te vragen en te manipuleren.

Wat is SQL?

Structured Query Language wordt gebruikt voor het manipuleren en definiëren van gegevens in databases. SQL biedt een gestandaardiseerde manier om specifieke vragen te stellen aan een database en gestructureerde query's te maken.

Een SQL-statement wordt geschreven om een antwoord uit een database te krijgen of om er een wijziging in aan te brengen. Query's verwijzen naar statements die vragen stellen. Er mogen spaties of witregels aan deze statements worden toegevoegd om het leesbaarder te maken.

Een SQL-statement bestaat uit clausules die met sleutelwoorden worden gedefinieerd. Daarnaast kan een statement veldnamen, tabelnamen en predicaten bevatten. Predicaten geven aan naar welke informatie of voorwaarde er wordt gezocht op zoek zijn, zoals:

SELECT first_name, last_name
    FROM quiz_results
    WHERE state_code = 'CA'
    ORDER BY last_name ASC;

Predicaten bevatten een waarde of voorwaarde die een expressie wordt genoemd. In het bovenstaande voorbeeld is 'CA' de expressie.

SQL-statements eindigen met een puntkomma en de gebruikte sleutelwoorden worden meestal in hoofdletters geschreven voor de leesbaarheid.

SQL wordt als een DML (Data Manipulation Language) gebruikt wanneer het de gegevens in de database manipuleert. SQL wordt als een DDL (Data Definition Language) gebruikt wanneer de structuur van databaseobjecten zoals tabellen, indexen en schema's te definiëren worden gewijzigd.

Handelingen waarbij data wordt gewijzigd, worden over het algemeen CRUD genoemd (Create, Read, Update, Delete).

Data opvragen uit een database

Data opvragen met SELECT

Sleutelwoord Uitleg Voorbeelden
SELECT Haalt gegevens op. 
SELECT company, first_name FROM people;
FROM Specificeert de tabel. 
SELECT last_name FROM people;

Het SELECT-statement wordt gebruikt om gegevens uit een database op te halen en maakt het mogelijk om gegevens te lezen. Het gebruik van SELECT is een veilige manier om gegevens te verkennen zonder deze daadwerkelijk in de database te wijzigen.

Daarnaast kan het SELECT-statement letterlijke tekst teruggeven door de gewenste tekst tussen enkele aanhalingstekens te plaatsen:

SELECT 'Hello World!';

De velden die worden opgehaald worden gedefinieerd tussen SELECT en FROM. Het is mogelijk om meerdere velden op te halen door de velden met een komma te scheiden. De volgorde van de velden in het SELECT-statement bepaalt tevens volgorde in de uitvoer.

Operator Uitleg Voorbeelden
Wild card (*) Selecteert alle velden in een tabel. 
SELECT * FROM people;

Het gebruik van het sterretje is handig voor het verkennen van data, maar wordt zelden gebruikt in een app of rapport doordat het veel informatie teruggeeft.

Query's verfijnen met WHERE

Sleutelwoord Uitleg Voorbeelden
WHERE Voegt een voorwaarde toe. 
SELECT * FROM people WHERE state_code = 'CA';

Met de WHERE-clausule kunnen voorwaarden aan een SELECT-statement worden toegevoegd, om resultaten te filteren op basis van specifieke criteria.

De volgorde van clausules in SQL is belangrijk. De WHERE-clausule moet na SELECT en FROM komen om geldig te zijn.

De tekst in de WHERE-clausule is hoofdlettergevoelig, wat betekent dat "CA" anders is dan "ca".

Meer criteria aan een statement toevoegen

Logische operatoren worden gebruikt om meerdere voorwaarden aan een WHERE-clausule toe te voegen.

Operator Uitleg Voorbeelden
AND Weergeeft een veld als alle condities true zijn. 
SELECT team, first_name
FROM people
WHERE state_code = 'CA' AND shirt_or_hat = 'shirt';
OR Weergeeft een veld als een conditie true is. 
SELECT team, first_name
FROM people
WHERE state_code = 'CA' OR shirt_or_hat = 'shirt';
!= Weergeeft een veld als de conditie false is. 
SELECT team, first_name
FROM people
WHERE state_code = 'CA' AND shirt_or_hat != 'shirt';
IS Gelijk aan. 
SELECT team, first_name
FROM people
WHERE state_code = 'CA' AND shirt_or_hat IS 'shirt';
IS NOT Niet gelijk aan. 
SELECT team, first_name
FROM people
WHERE state_code = 'CA' AND shirt_or_hat IS NOT 'shirt';
<> Niet gelijk aan. 
SELECT team, first_name
FROM people
WHERE state_code = 'CA' AND shirt_or_hat <> 'shirt';

Haakjes kunnen worden gebruikt om voorwaarden te groeperen en de volgorde van evaluatie te bepalen, vergelijkbaar met het gebruik ervan in algebra.

SELECT shirt_or_hat, state_code, first_name, last_name
FROM people
WHERE (state_code = 'CA' OR state_code = 'CO') AND shirt_or_hat IS 'shirt';

Het onjuist groeperen van voorwaarden is een veelgemaakte fout die kan leiden tot onverwachte resultaten.

Reacties verbreden en limiteren

Soms hoeft slechts een deel van het veld met een waarde overeen te komen in plaats van het gehele veld, zoals de eerste letter van een straat. In deze gevallen kan de LIKE-operator worden gebruikt.

Operator Uitleg Voorbeelden
LIKE Is true als het veld met het gespecificeerde patroon overeenkomt. 
SELECT * FROM people
WHERE state_code LIKE 'C%';

De LIKE operator maakt gebruik van '%' om delen van een veld te matchen. 'C%' betekent dat alle velden die met een 'C' beginnen gekwalificeerd zijn, terwijl '%C' de velden die met 'C' eindigen kwalificeert. "%C%" kwalificeert velden die een 'C' bevatten en 'B%N' de velden die met een 'B' beginnen en met een 'N' eindigen.

De LIKE-operator is standaard niet hoofdlettergevoelig, maar dit kan afhangen van de database- en kolominstellingen.

Sleutelwoord Uitleg Voorbeelden
LIMIT Specificeert hoeveel resultaten maximaal worden weergeven. 
SELECT * FROM people
WHERE state_code LIKE 'C%'
LIMIT 5;
OFFSET Specificeert hoeveel velden worden overgeslagen voordat het resultaat wordt weergeven. 
SELECT * FROM people
WHERE state_code LIKE 'C%'
LIMIT 5 OFFSET 5;

Reacties organiseren met ORDER BY

Sleutelwoord Uitleg Voorbeelden
ORDER BY Specificeert hoe resultaten worden gesorteerd. 
SELECT first_name, last_name 
FROM people
ORDER BY first_name DESC;

Standaard wordt er oplopend gesorteerd (ASC). Dit kan naar aflopend worden veranderd met DESC.

Het is mogelijk om op meerdere velden te sorteren, bijvoorbeeld:

SELECT state_code, last_name, first_name 
FROM people
ORDER BY state_code, last_name DESC;

In het bovenstaande wordt de state_code eerst alfabetisch gesorteerd en wordt last_name vervolgens in aflopende volgorde gesorteerd.

Het sorteren van gegevens is handig voor rapportages en toepassingen, zoals het weergeven van gevallen van klantenondersteuning op datum of ticket-ID.

Informatie over de data vinden

Sommige functies worden gebruikt om meer te weten te komen over de gegevens zelf, in plaats van alleen records terug te geven die overeenkomen met query's.

Functie Uitleg Voorbeelden
LENGTH Specificeert de hoeveelheid karakters in een veld. 
SELECT first_name, LENGTH(first_name)
FROM people;
DISTINCT Haalt alleen de unieke waardes op. 
SELECT DISTINCT(first_name)
FROM people;
COUNT Weergeeft hoeveel velden aan een specifiek criteria voldoen. 
SELECT COUNT(*)
FROM people
WHERE state_code = 'CA';

Data opvragen uit twee of meer tabellen

Data opvragen uit twee of meer tabellen

Het is mogelijk om een query te beantwoorden met data uit meer dan één tabel door middel van een gemeenschappelijk stuk informatie.

Sleutelwoord Uitleg Voorbeelden
JOIN Voegt data uit meerdere tabellen samen op basis van een gerelateerde kolom. 
SELECT people.first_name, people.state_code, states.division
FROM people
JOIN states ON people.state_code = states.state_abbrev;

Het is een best practice om met puntnotatie aan te geven bij welke tabel een veld hoort. Dit voorkomt errors in het geval dat de tabellen een overeenkomende kolomnaam delen.

Het is mogelijk om gegevens uit verschillende tabellen op te vragen zonder het JOIN-sleutelwoord te gebruiken, wat een impliciete join wordt genoemd.

SELECT ppl.first_name, st.state_name
FROM people ppl, states st
WHERE ppl.state_code = st.state_abbrev;

Het is meestal het beste om de expliciete join syntax te gebruiken, om alles duidelijk te houden. Daarnaast kan een impliciete join ook tot onbedoelde gevolgen leiden, zoals een cross join.

Aliassen kunnen aan een tabel worden toegewezen om query's te vereenvoudigen en leesbaarder te maken, vooral bij lange tabel namen.

JOIN types begrijpen

Met een JOIN wordt data tussen verschillende tabellen aan elkaar geassocieerd. Dit kan op verschillende manieren worden bereikt:

Sleutelwoord Uitleg Voorbeelden
CROSS JOIN Combineert elke record van de eerste tabel met alle records van de tweede tabel. 
SELECT * FROM people CROSS JOIN states;
INNER JOIN Voegt data uit meerdere tabellen samen op basis van een gerelateerde kolom. 
SELECT * FROM people INNER JOIN states 
ON people.state_code = states.state_abbrev;
LEFT JOIN Voegt data uit meerdere tabellen samen op basis van een gerelateerde kolom en weergeeft alle linker waarden. 
SELECT * FROM people LEFT JOIN states 
ON people.state_code = states.state_abbrev;
RIGHT JOIN Voegt data uit meerdere tabellen samen op basis van een gerelateerde kolom en weergeeft alle rechter waarden. 
SELECT * FROM people RIGHT JOIN states 
ON people.state_code = states.state_abbrev;
FULL OUTER JOIN Combineert twee tabellen op basis van een gerelateerde kolom en geeft de overlappende en niet overlappende waarden. 
SELECT * FROM people FULL OUTER JOIN states 
ON people.state_code = states.state_abbrev;

Een INNER JOIN geeft alleen de overlappende records terug. Het kan echter voortkomen dat een record geen overeenkomende sleutel waarde heeft in de andere tabel. De verschillende vormen van OUTER JOIN worden gebruikt, om deze waarden alsnog te kunnen weergeven. De niet records die geen overeenkomst hebben met records uit de andere tabel worden in dit geval aan NULL-waarden verbonden.

Sommige van de JOIN vormen die beschikbaar zijn in SQL worden niet in alle software ondersteunt. MySQL biedt bijvoorbeeld geen ondersteuning voor de FULL OUTER JOIN, terwijl SQLite de RIGHT JOIN en de FULL OUTER JOIN niet ondersteund.

Resultaten groeperen

Sleutelwoord Uitleg Voorbeelden
GROUP BY Groepeert records op basis van de kolom waarde. 
SELECT first_name, COUNT(first_name)
FROM people
GROUP BY first_name;

De bovenstaande query laat zien hoe vaak een voornaam in de database voor komt. Zonder het GROUP BY-sleutelwoord, worden alleen de voornaam van de laatste record en de totale hoeveelheid voornamen in een database weergeven:

De gegroepeerde velden moeten op elkaar aansluiten, om onbruikbare gegevens te voorkomen zoals hieronder het geval is:

SELECT first_name, COUNT(first_name)
FROM people
GROUP BY last_name;

Hier wordt het aantal voornamen per achternaam gegroepeerd, waarbij alleen de eerste voornaam per achternaam zichtbaar is, wat niet zinvol is.

Het mogelijk om meerdere velden te groeperen om gedetailleerde resultaten te verkrijgen. De onderstaande query weergeeft hoe vaak een bepaald aantal quizpunten is behaald per staat:

SELECT state_code, quiz_points, COUNT(quiz_points)
FROM people
GROUP BY state_code, quiz_points;

Niet elke implementatie van SQL biedt ondersteuning voor alle verschillende datatypes. Het is daarom van belang om de documentatie van de SQL-tools door te nemen.

Datatype Toelichting

Berekeningen in SQL

SQL ondersteunt aritmetische bewerkingen zoals optellen, aftrekken, vermenigvuldigen, delen en modulo:

Operator Betekenis Voorbeeld
+ Optellen SELECT 1 + 1; //2
- Aftrekken SELECT 3 - 2; //1
* Vermenigvuldigen SELECT 7 * 9; //63
/ Delen SELECT 68 / 4; //17
% Modulo SELECT 5 % 2; //1

SQL neemt aan dat bewerkingen integers gebruiken, behalve als er bijvoorbeeld expliciet wordt gespecificeerd dat een float wordt gebruikt. Dit is van belang aangezien de precisie van de resultaten kan worden beïnvloed, wat niet wenselijk is bij financiële of wetenschappelijke berekeningen.

Daarnaast ondersteunt SQL logische vergelijkingsoperatoren die het mogelijk maken om gegevens te filteren op basis van voorwaarden.

Operator Betekenis
> Groter dan.
< Kleiner dan.
>= Groter of gelijk aan.
<= Kleiner of gelijk aan.
= Gelijk aan.
!= Niet gelijk aan.
<> Niet gelijk aan.

SQL biedt functies zoals MAX, MIN, SUM en AVG om geaggregeerde berekeningen uit te voeren op numerieke velden.

Sleutelwoord Uitleg Voorbeelden
MAX Retourneert de grootste waarde. 
SELECT MAX(quiz_points)
FROM people;
MIN Retourneert de kleinste waarde. 
SELECT MIN(quiz_points)
FROM people;
SUM Retourneert een optelling van alle waarden. 
SELECT SUM(quiz_points)
FROM people;
AVG Retourneert het gemiddelde van alle waarden. 
SELECT team, COUNT(*), SUM(quiz_points), AVG(quiz_points)
FROM people
GROUP BY team;

Compound Select

Een subquery of sub-selectie is een secundaire SELECT-functie binnen een query, die wordt gebruikt om gegevens voor de hoofdquery verder te specificeren:

SELECT first_name, last_name, quiz_points
FROM people
WHERE quiz_points = (SELECT MAX(quiz_points) FROM people);

Geaggregeerde functies zoals MAX, MIN en AVG functioneren niet als een onderdeel van een WHERE-functie en leiden normaliter tot een error. Daarom maakt het bovenstaande voorbeeld gebruik van een subquery om fouten te voorkomen.

Een subquery kan ook worden ingezet om de informatie uit een tabel als informatie voor een zoekopdracht in een andere tabel te gebruiken.

SELECT *
FROM people
WHERE state_code = (SELECT state_abbrev FROM states WHERE state_name = 'Minnesota');

Data transformeren

Met SQL kan de data uit de database worden getransformeerd aan de hand van query's:

Sleutelwoord Uitleg Voorbeelden
LOWER Transformeert de karakters in de string naar kleine letters. 
SELECT LOWER(first_name), last_name
FROM people;
UPPER Transformeert de karakters in de string naar hoofdletters. 
SELECT first_name, UPPER(last_name)
FROM people;
SUBSTR(ING) Extraheert karakters uit een string. 
SELECT first_name, SUBSTR(last_name, 1, 5)
FROM people;

Het SUBSTRING-sleutelwoord bevat twee additionele parameters naast een veldnaam: Een nummer dat aangeeft vanaf welk karakter er wordt geteld en een nummer die het aantal karakters representeert. In het bovenstaande voorbeeld worden de eerste vijf karakters geëxtraheerd.

De parameter die aangeeft vanaf welk nummer wordt geteld, mag een negatief getal zijn. Dit geeft aan dat er vanaf het einde van de string wordt geteld.

Sleutelwoord Uitleg Voorbeelden
REPLACE Vervangt tekst in een string. 
SELECT REPLACE(first_name, "a", "-")
FROM people;

In het bovenstaande voorbeeld vervangt het REPLACE-sleutelwoord alle kleine letters "a" met een "-". Hoofdletters worden niet beïnvloed.

Sleutelwoord Uitleg Voorbeelden
CAST Converteert een datatype naar een ander datatype. 
SELECT quiz_points
FROM people
ORDER BY CAST(quiz_points AS CHAR);

Het CAST-sleutelwoord komt van pas als er geen controle is over het schema van de database of als de manier waarop gegevens worden opgeslagen niet kan worden gewijzigd. Het AS-sleutelwoord wordt gebruikt om de gewenste datatype te specificeren.

Aliassen creëren met AS

Sleutelwoord Uitleg Voorbeelden
AS Hernoemt de naam van een veld 
SELECT first_name AS firstname, UPPER(last_name) AS surname
FROM people
WHERE firstname = 'Laura';

Het gebruik van aliassen kan de queryresultaten verduidelijken, wat wenselijk kan zijn als de data direct naar een spreadsheet of een toepassingen wordt geëxporteerd.

De gespecificeerde aliasnaam kan tevens als plaatsvervanger worden gebruikt voor de originele veldnaam binnen de query.

Een aliasnaam hoort geen spaties te bevatten en wanneer spaties nodig zijn worden apostroffen (") gebruikt.

Data toevoegen of aanpassen

Data toevoegen aan een tabel

Sleutelwoord Uitleg Voorbeelden
INSERT Voegt nieuwe data toe aan een bestaande tabel. 
INSERT INTO people (first_name)
VALUES ('Bob');

In het bovenstaande voorbeeld is er een waarde voor één veld opgegeven. De overige velden die leeg zijn gelaten krijgen een waarde van NULL als plaatsvervanger.

Het is mogelijk om meer dan één record tegelijk toe te voegen door de veldnamen en de reeks van gewenste data na het VALUES-sleutelwoord op te geven:

INSERT INTO people
(first_name, last_name)
VALUES
('George', 'White'),
('Jenn', 'Smith'),
('Carol', NULL);

De bovenstaande NULL-waarde zal niet in elk veld worden geaccepteerd. Soms hebben tabellen vastgestelde regels die dicteren dat gegevens in een bepaald formaat moeten worden aangeleverd. Er kan bijvoorbeeld worden bepaald dat een specifiek veld een unieke waarde moet bevatten of geen NULL-waarde mag hebben.

Daarnaast kan er in een kolom worden vastgesteld dat er automatisch nieuwe waarden voor nieuwe records worden gegenereerd.

In de DB Browser worden wijzigingen pas permanent gemaakt nadat er op "Write Changes" wordt geklikt, waardoor er zonder de werkelijke gegevens te beïnvloeden kan worden geoefend.

Data aanpassen in een tabel

Sleutelwoord Uitleg Voorbeelden
UPDATE Wijzigt bestaande gegevens in een tabel. 
UPDATE people
SET first_name = 'Bob', last_name = 'Morrison'
WHERE id_number = 175;

Het WHERE-sleutelwoord wordt gebruikt om de te bewerken records te specificeren. Zonder WHERE zouden de wijzigingen op alle records in de tabel worden toegepast.

Het is van belang om zo specifiek mogelijk te zijn tijdens wijzigen van records, zodat onbedoelde wijzigingen kunnen worden voorkomen. Het gebruik van het SELECT-sleutelwoord, additionele velden en unieke identificeerders zoals ID-nummers kunnen helpen om precies de juiste record te vinden.

Data verwijderen uit een tabel

Sleutelwoord Uitleg Voorbeelden
DELETE Verwijdert een record uit een tabel. 
DELETE FROM people 
WHERE id_number = 1001;

Net als bij het aanpassen van data is het van belang om specifiek te zijn tijdens het verwijderen van gegevens. Het WHERE-sleutelwoord in het bovenstaande voorbeeld voorkomt dat alle records uit de people tabel worden verwijderd.

Het SELECT-sleutelwoord wordt gebruikt om de voorwaarde te testen, zodat deze op de juiste records is gericht.

Bij het verwijderen van NULL-waarden worden IS NULL of IS NOT NULL in plaats van operatoren gebruikt:

DELETE FROM people WHERE quiz_points IS NULL;
