Learning C#

Overzicht van C#

Hello World C#

Het traditionele "Hello World"-voorbeeld helpt bij het leren van een nieuwe programmeertaal. In dit geval wordt een C# console-applicatie gecreÃ«erd met behulp van de dotnet-CLI en worden de minimale programmastructuur, invoer- en uitvoermethoden, en de mogelijkheid om boilerplate-code te verminderen via toplevel statements belicht.

In de terminal wordt naar de volgende map genavigeerd:

cd start/overview/hello-world

Vervolgens wordt het volgende commando uitgevoerd, om een nieuwe console-app te genereren:

dotnet new console

Hiermee wordt een .csproj-projectbestand, een lege obj-map en een program.cs met auto-gegenereerde code aangemaakt. De gegenereerde program.cs ziet er als volgt uit:

using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello World!");
        }
    }
}

"Using System;" importeert de System-namespace en "namespace HelloWorld" voorkomt naamconflicten met andere libraries. Alle methoden komen in "class Program" en "static void Main(string[] args)" is het entry-point van de applicatie. Ten slot wordt het geheel verplicht met het ";" teken afgesloten.

Met de Console-klasse kan tekst naar de terminal worden overgebracht en de invoer worden gelezen:

Console.WriteLine("What is your name?");
string str = Console.ReadLine();
Console.WriteLine("Why hello there " + str);
Methode Uitleg Voorbeeld
WriteLine(text) Schrijft een regel tekst naar de console 
Console.WriteLine("Wat is je naam?");
ReadLine() Leest een regel tekst van de gebruiker 
string naam = Console.ReadLine();

Om het programma uit te voeren wordt in de terminal het volgende getypt:

dotnet run

Dit bouwt en start de applicatie. Na invoer van de naam verschijnt de aangepaste groet. Om boilerplate te verminderen, kunnen ten slot toplevel statements worden gebruikt. Hierdoor blijven alleen de statements over:

Console.WriteLine("Hello World!");
Console.WriteLine("Wat is je naam?");
string naam = Console.ReadLine();
Console.WriteLine("Why hello there, " + naam);

In C# wordt vóór de variabele naam het datatype vermeld, wat helpt fouten tijdens het compileren te voorkomen. Hieronder staan de belangrijkste datatypes in C# beschreven:

Het sleutelwoord var kan gebruikt worden om de compiler het type te laten afleiden.

Arrays worden gedeclareerd met [] en kunnen meteen met waarden worden geïnitialiseerd:

De impliciete conversie vindt plaats tussen datatypes die verenigbaar zijn zoals int naar long. Null-coalescing maakt het mogelijk om met null-variabelen te werken door standaardwaarden of toekenningen te bieden:

C# Commentaar schrijven

C# biedt drie verschillende manieren om commentaar om aan code toe te voegen, om daarmee de code te verduidelijken en documentatie te genereren: single-line, multi-line en XML-documentatie commentaar.

Single-line commentaar begint met //, loopt tot het einde van de regel en is daarmee geschikt voor korte toelichtingen bij één regel of directe aantekeningen:

// Dit is een single-line comment
int x = 5;  // Variabele x wordt op 5 gezet

Multi-line commentaar begint met /* en eindigen met */ en is ideaal voor langere toelichtingen of voor het tijdelijk uitschakelen van codeblokken:

/*
    Dit is een multi-line comment.
    Hij kan meerdere regels beslaan
    zonder telkens // te gebruiken.
*/
int y = 10;

XML-documentatie commentaar begint met /// en is opgebouwd uit XML-tags. Dit commentaar wordt door het .NET-bouwproces geëxtraheerd voor automatische documentatie:

/// <summary>
/// De hoofd­methode van de sample applicatie.
/// </summary>
/// <param name="args">
/// Array van string-argumenten van de commandline.
/// </param>
/// <returns>
/// Geen returnwaarde.
/// </returns>
static void Main(string[] args)
{
    // implementatie…
}

Een aantal van de meest gebruikte tags die in XML-documentatie commentaar worden gebruikt, staan hieronder weergeven:

Tag Uitleg
&lt;summary&gt; Beschrijft wat een methode of klasse doet.
&lt;param&gt; Uitleg per parameter, met attribuut name="..."
&lt;returns&gt; Omschrijft de returnwaarde

Om automatisch documentatie te genereren op basis van XML-documentatie commentaar wordt eer het project­bestand aangepast (.csproj):

&lt;PropertyGroup&gt;
    &lt;GenerateDocumentationFile&gt;true&lt;/GenerateDocumentationFile&gt;
    &lt;DocumentationFile&gt;Comments.xml&lt;/DocumentationFile&gt;
&lt;/PropertyGroup&gt;

En vervolgens wordt de build uitgevoerd, waardoor er een Comments.xml verschijnt met alle XML-comments, klaar voor gebruik in documentatietools:

dotnet build

C# Programma flow

Met behulp van een if-statement en de optionele else if-takken is het mogelijk om beslissingen in C#-programma's te nemen:

Sleutelwoord Uitleg Voorbeeld
If Voert bij overeenkomst de code ut. 
int theVal = 55;

if (theVal == 50)
{
    Console.WriteLine("theVal is 50");
}
else if Voert de code uit als de voorgaande voorwaarde(s) onwaar is én de eigen voorwaarde waar is 
int time = 22;

if (time < 10) 
{
    Console.WriteLine("Good morning.");
} 
else if (time < 20) 
{
    Console.WriteLine("Good day.");
} 
else 
{
    Console.WriteLine("Good evening.");
}
else Definieert het gedrag als alle voorgaande voorwaarden onwaar zijn. 
int time = 20;
if (time < 18)

{
    Console.WriteLine("Good day.");
} 
else 
{
    Console.WriteLine("Good evening.");
}

Er kan een ongelimiteerde hoeveelheid extra voorwaarden worden toegevoegd met else if, maar bij meer dan ~4–5 takken wordt het onoverzichtelijk.

De ternary-operator is een verkorte voorwaarde die de if–else-constructies verkleint tot één expressie. Het ziet er bijvoorbeeld als volgt uit:

resultaat = (conditie) ? waardeBijTrue : waardeBijFalse;

Voorwaardelijke logica met "switch"

Met behulp van een switch-statement is het mogelijk om beslissingen in C#-programma's te nemen. Een switch statement wordt toegepast wanneer het aantal vergelijkingen hoog is, omdat het in deze gevallen de leesbaarheid en onderhoudbaarheid verbeterd. De if–else statement is doorgaans overzichtelijker, als de hoeveelheid vergelijkingen lager is dan vier.

Sleutelwoord Uitleg Voorbeeld
switch Evalueert éénmalig de expressie. 
int val = 3;
switch (theVal)
{
    case 50:
        Console.WriteLine("theVal is 50");
        break;
    case 51:
        Console.WriteLine("theVal is 51");
        break;
    default:
        Console.WriteLine("theVal is something else");
        break;
}
case Voert bij overeenkomst de code tot de volgende break uit. 
int day = 4;
switch (day) 
{
case 1:
    Console.WriteLine("Monday");
    break;
case 2:
    Console.WriteLine("Tuesday");
    break;
}
break Verlaat de switch-block, zodat volgende cases niet meedraaien. 
int day = 4;
switch (day) 
{
    case 7:
    Console.WriteLine("Sunday");
    break;
}
default Definieert het gedrag van alle niet-afgehandelde gevallen in de switch statement (vergelijkbaar met else). 
int day = 4;
switch (day) 
{
    case 7:
        Console.WriteLine("Today is Sunday.");
        break;
    default:
        Console.WriteLine("Looking forward to the Weekend.");
        break;
}

Het is tevens mogelijk om met meerdere labels naar dezelfde code te verwijzen door de cases opeenvolgend achter elkaar te plaatsen. Dit kan er bijvoorbeeld als volgt uitzien:

case 52:
case 53:
case 54:
    Console.WriteLine("theVal is between 52 and 54");
    break;

For loops

In C# kunnen herhalende bewerkingen worden uitgevoerd met de for– en foreach–loop. De for-loop is bedoeld voor een vooraf bepaald aantal iteraties, terwijl foreach ideaal is voor het doorlopen van verzamelingen. Een standaard for-loop ziet er als volgt uit:

int myVal = 15;

for (int i = 0; i < myVal; i++)
{
    Console.WriteLine("i is currently {0}", i);
}

In het bovenstaande voorbeeld wordt de startwaarde van de teller geinitialiseerd met (int i = 0):, terwijl (i < myVal): de voorwaarde is waaronder de loop blijft draaien zolang de conditie True is. Met (i++): wordt aangegeven dat de loop opnieuw wordt uitgevoerd na elke rondgang (telt i met 1 op).

Een foreach–loop herhaalt voor elk element in de collectie. Voor verzamelingen (arrays, strings, lijsten) is de foreach-loop daarom eenvoudiger en minder foutgevoelig. Een standaard foreach–loop kan er als volgt uitzien:

int[] nums = { 3, 14, 15, 92, 6 };

foreach (int num in nums)
{
    Console.WriteLine("num is currently {0}", num);
}

// Ook op strings toepasbaar, bijvoorbeeld om 'O'-tekens te tellen:
string str = "Hello, World! C# for loops";
int count = 0;

foreach (char c in str)
{
    if (c == 'O' || c == 'o')
        count++;
}

Console.WriteLine("Aantal O-tekens: {0}", count);

While loops

Met while loops kan een blok code worden herhaald zolang de gedefinieerde voorwaarde waar is. De voorwaarde waarop wordt getoetst staat bovenaan en wordt vóór elke iteratie geëvalueerd. Daarnaast heeft een while-loop een stopconditie, waarmee de loop wordt beëindigd:

string inputStr = "";

while (inputStr != "exit")
{
    inputStr = Console.ReadLine();
    Console.WriteLine("You entered: {0}", inputStr);
}

De do-while-lus is een variant van de while loop die minimaal één iteratie garandeert. Bij deze variant staat de voorwaarde onderin en deze loopvorm wordt altijd minimaal één keer uitgevoerd. Daarna is deze loop alleen doorlopend als de voorwaarde waar blijft. Een standaard while loop ziet er bijvoorbeeld zo uit:

string inputStr = "";

do
{
    inputStr = Console.ReadLine();
    Console.WriteLine("You entered: {0}", inputStr);
}
while (inputStr != "exit");

Break en continue gebruiken

De uitvoering van loops in C# nauwkeuriger worden aangestuurd met de break en continue statements. Break wordt toegepast om de code te stoppen wanneer de eerste waarde die aan een criterium voldoet, terwijl continue tijdelijk ongewenste waarden negeert en de rest wel verwerkt:

Sleutelwoord Uitleg Voorbeeld
break Stopt de volledige lus direct, ongeacht van de oorspronkelijke stopvoorwaarde. 
int[] values = { 15, 7, 12, 23, 41, 28, 9 };

foreach (int val in values)
{
    Console.WriteLine(val);
    if (val >= 40)
        break;  // bij eerste waarde ≥ 40, lus beëindigen
}
continue Slaat de rest van de iteratie over als de gedefinieerde voorwaarde wordt voldaan en gaat verder met de volgende iteratie in de lus. 
int[] values = { 15, 7, 12, 23, 41, 28, 9 };

foreach (int val in values)
{
    if (val >= 20 && val <= 29)
        continue;  // sla waarde in de jaren '20 over

    Console.WriteLine(val);
}

Uitzonderingen

Het afhandelen van uitzonderingen verwijst naar het opvangen en afhandelen van fouten zonder dat de gebruiker een onbegrijpelijke foutmelding ziet. Eén van de manieren waarmee fout afhandeling kan worden verwezenlijkt, is het gebruik van try, catch, throw en finally:

Statement Uitleg Voorbeeld
try Omringt de code die een exception kan opleveren. 
try
{
    ProcessShapes(shapeAmount);
}
catch (Exception e)
{
    LogError(e, "Shape processing failed.");
    throw;
}
catch Vangt één type uitzondering op; kan meerdere catch-blokken hebben. 
try
{
    int[] myNumbers = {1, 2, 3};
    Console.WriteLine(myNumbers[10]);
}
catch (Exception e)
{
    Console.WriteLine("Something went wrong.");
}
throw Creert handmatig een uitzondering 
string first = args.Length >= 1 
    ? args[0]
    : throw new ArgumentException("Please supply at least one argument.");
finally Wordt altijd uitgevoerd, ongeacht of een exception optrad of niet. 
try
{
    int[] myNumbers = {1, 2, 3};
    Console.WriteLine(myNumbers[10]);
}
catch (Exception e)
{
    Console.WriteLine("Something went wrong.");
}
finally
{
    Console.WriteLine("The 'try catch' is finished.");
}

Het is tevens mogelijk om meerdere catch-blokken toe te voegen, zodat verschillende fouttypes anders worden af gehandeld:

try
{
    int x = 1500;
    if (x > 1000)
        throw new ArgumentOutOfRangeException(
            "x", "x must be 1000 or less");
}
catch (DivideByZeroException e)
{
    Console.WriteLine("Delen door nul: " + e.Message);
}
catch (ArgumentOutOfRangeException e)
{
    Console.WriteLine("Waarde te hoog: " + e.ParamName);
    Console.WriteLine(e.Message);
}
catch (Exception e)  // generieke handler voor alle andere exceptions
{
    Console.WriteLine("Algemene fout: " + e.Message);
}
finally
{
    Console.WriteLine("Opruimcode of afsluitmeldingen hier.");
}

C# Strings

String operaties

Strings bieden een set aan eigenschappen en methoden om tekst te verwerken en worden in C# als objecten behandeld. Strings bieden daarnaast directe toegang tot lengte en individuele karakters, én kunnen als een sequentie worden doorlopen.

Met het Length-eigenschap kan het aantal tekens in de string worden teruggegeven, terwijl indexering toegang tot een specifiek teken biedt via vierkante haken ([]), beginnend bij index 0. Met een foreach(char ch in str1)-lus kunnen alle karakters één voor één worden verwerkt.

Strings kunnen met elkaar worden samengevoegd of juist van een scheidingsteken worden voorzien, doormiddel van de volgende methodes:

Methode Uitleg Voorbeeld
Concat() Voegt een of meer strings aan elkaar tot één geheel. 
outStr = String.Concat(strs);
Join() Voegt meerdere strings samen met een opgegeven scheidingsteken of -reeks. 
outStr = String.Join(".", strs); // "een.twee.drie"
outStr = String.Join("---", strs); // "een---twee---drie"

Daarnaast biedt C# statische en instantie-methoden om strings op uiteenlopende manieren te vergelijken:

Methode Uitleg Voorbeeld
Compare() Vergelijkt de waarden volgens de Unicode-waarde. Retourneert <0 als waarde a vóór b komt, 0 als het gelijk is en >0 als a ná b komt. 
int cmp = String.Compare(str2, "This is a string");
Console.WriteLine(cmp); // 0
Equals() Controleert of twee strings exact gelijk zijn (bool). 
bool isEqual = str2.Equals(str3);
Console.WriteLine(isEqual); // false (hoofdletterverschil)

Ook bestaan er speciale methoden waarmee tekst in een string kan worden opgezocht en (gedeeltelijk) kan worden vervangen:

Methode Uitleg Voorbeeld
IndexOf() Geeft de index van de eerste instantie van een waarde, of –1 als deze niet voorkomt. 
int idxE = str1.IndexOf('e');
int idxFox = str1.IndexOf("fox");
LastIndexOf() Geeft de index van de laatste instantie van een waarde, of –1 als deze niet voorkomt. 
int lastE = str1.LastIndexOf('e');
int lastThe = str1.LastIndexOf("the");
Replace() Vervangt alle instanties van een waarde met de nieuwe waarde en retourneert een nieuwe string. 
string replaced = str1.Replace("fox", "cat");
Console.WriteLine(replaced); // "The quick brown cat …"
Console.WriteLine(replaced.IndexOf("fox")); // -1

String formateren

De functie voor samengestelde .NET-opmaak neemt een lijst met objecten en een tekenreeks met samengestelde opmaak als invoer. In C# biedt de samengestelde opmaak-syntax uitgebreide controle over de weergave van variabelen in een string. De algemene structuur hiervoor is:

{index[,alignment][:formatSpecifier[precision]]}

Standaardtekenreeksen voor numerieke notatie worden gebruikt om algemene numerieke typen op te maken. Een standaardtekenreeks voor numerieke notatie heeft de volgende notatie:

[format specifier][precision specifier]

In de onderstaande tabel worden de standaardaanduidingen voor numerieke notaties beschreven:

Standaardtekenreeks Beschrijving Voorbeeld
D Decimaal (integer) 
{0:D4} → 0007
N Getal met duizendtalscheiding 
{0:N} → 1 234,00
F Fixed-point (vast aantal decimalen) 
{0:F2} → 1234,00
G General (kortste notatie) 
{0:G} → 1234
E Exponentiële notatie 
{0:E3} → 1,234E+003
C Valuta (locale settings) 
{0:C} → € 1 234,00
P Procent (vermenigvuldigd met 100%) 
{0:P0} → 12 345 678 %

Na de standaardtekenreeks kan een geheel getal volgen om het aantal cijfers/de breedte aan te geven, bijvoorbeeld:

Format Uitleg Output
D6 Minimaal 6 cijfers (leading zeros) 001234
N2 2 cijfers na de komma, duizendtalscheiding 1 234,57
F1 Vast 1 cijfer na de komma 1234,6
G3 Maximaal 3 significante cijfers, anders expo 1,23E+03

Ten slot kan met de alignment-optie de kolombreedte worden bepaald:

string[] quarters = { "Q1", "Q2", "Q3", "Q4" };

Console.WriteLine(
    "{0,12}{1,12}{2,12}{3,12}",
    quarters[0], quarters[1], quarters[2], quarters[3]);

String interpolatie

Doormiddel van stringinterpolatie kunnen variabelen en expressies direct in een string worden opgenomen door een $ vóór de openingsquote te plaatsen. Binnen de string worden variabelen tussen accolades {} gezet, zonder gebruik van index-notatie, zoals:

Console.WriteLine($"Auto: {year} {make} {model}, {miles} miles, {price}");

Alle standaard format-specifiers kunnen binnen interpolatie gebruikt worden na een dubbele punt en complexere berekeningen of methode-calls kunnen direct binnen de {} worden uitgevoerd:

Console.WriteLine($"Afstand: {miles * 1.6:F2} km");

Om een letterlijke accolades in de uitvoer op te nemen, worden dubbele accolades gebruikt:

Console.WriteLine($"Afstand: {{ {miles} }} miles");

De StringBuilder gebruiken

In .NET is string onveranderlijk waardoor er bij bewerkingen telkens een nieuwe string ontstaat. Bij veel concatenaties gaat dit echter ten koste gaat van prestaties en geheugen. De StringBuilder-klasse (in System.Text) biedt een efficiënte buffer voor het opbouwen en manipuleren van tekst zonder telkens nieuwe string-instanties aan te maken.

using System.Text;

var sb = new StringBuilder(
    "The quick brown fox", // initiële inhoud
    200                    // initiële capaciteit
);

Console.WriteLine($"Capacity: {sb.Capacity}, Length: {sb.Length}");

De belangrijkste methoden die bij de StingBuilder worden toegepast staan hieronder weergeven:

Methode Uitleg Voorbeeld
sb.Append(string) Voegt achteraan tekst toe zonder regeleinde 
sb.Append(" jumped over the lazy dog.");
sb.AppendLine(string) Zelfde als Append, plus een CRLF 
sb.AppendLine("And then he ran away.");
sb.AppendFormat(fmt,…) Voegt een geformatteerde string toe (index-notatie) 
sb.AppendFormat("He did this {0} times.", jumpCount);
sb.AppendJoin(sep,…) Voegt een reeks samen met een scheidingsteken 
sb.AppendJoin(", ", new[] { "goats","cats","pigs" });
sb.Replace(old, new) Vervangt alle voorvallen van old door new 
sb.Replace("fox", "cat");
sb.Insert(idx, text) Voegt text in op positie idx 
sb.Insert(0, "This is the ");
sb.ToString() Converteert de opgebouwde inhoud naar een reguliere string 
string result = sb.ToString(); Console.WriteLine(result);

Strings parsen

Het omzetten van tekstuele getallen of booleans naar native C#-typen is een veelvoorkomende taak. Elk van de standaard datatypen beschikt over een Parse-methodes die hierbij directe ondersteuning biedt.

int    i = int.Parse("123");
float  f = float.Parse("3.14");
bool   b = bool.Parse("True");
decimal d = decimal.Parse("1,234.56");

Wanneer een Parse-methodes een ongeldige invoer verwerkt, ontstaat er een FormatException. Daarom wordt er doorgaans een gebruik gemaakt van een try–catch-blok, zoals:

try
{
    int i = int.Parse(numStr1);
    Console.WriteLine(i);
}
catch (FormatException)
{
    Console.WriteLine("Ongeldige notatie");
}

Met NumberStyles kan worden aangeven welke opmaak de parser mag accepteren (bijvoorbeeld duizendtallen of decimalen) en bevindt zich in System.Globalization:

Methode Uitleg Voorbeeld
NumberStyles.Integer Neemt alleen gehele getallen "1234"
NumberStyles.Float Neemt alleen getallen met een komma of punt voor decimalen "3.0"
NumberStyles.AllowThousands Neemt alleen getallen met een komma of punt als duizendtalscheiding "1,234"

TryParse geeft true of false terug en gebruikt een out-parameter voor het resultaat, zodat er geen try–catch nodig is:

string input = "123";
if (Int32.TryParse(input, out int result))
{
    Console.WriteLine($"Parsed: {result}");
}
else
{
    Console.WriteLine("Parse mislukt");
}

C# Functies

Basisprincipes van functies

Functies, oftewel methoden, bundelen herbruikbare code in logisch afgebakende blokken. Een functie kent een returntype, een naam, een optionele parameterlijst en een body tussen accolades, zoals:

float MilesToKm(float miles)
{
    float result = miles * 1.6f;
    return result;
}

Console.WriteLine($"8 mi = {MilesToKm(8.0)} km");
Console.WriteLine($"52 mi = {MilesToKm(52.0)} km");

Met het returntype geeft de functie de gegevenstype van de waarde terug. Bij een functie zonder returnwaarde zijn de parameters optioneel, wordt er geen waarde teruggegeven en is er geen return nodig. Dit ziet er als volgt uit:

void PrintWithPrefix(string theStr)
{
    Console.WriteLine($"[PREFIX]: {theStr}");
}

PrintWithPrefix("Test String");
PrintWithPrefix("Another Test String");

Benaamde en standaard parameters

C# ondersteunt standaard parameters en benaamde argumenten waarmee de leesbaarheid en flexibiliteit van functieroepen kan worden verhoogd. Bij het definiëren van een parameter kan een standaardwaarde worden opgegeven. Als de argumentwaarde bij het aanroepen wordt weggelaten, dan geldt deze default.

Bij het definiëren van een parameter kan een standaardwaarde worden opgegeven. Als de argumentwaarde bij het aanroepen wordt weggelaten, dan geldt deze default:

void PrintWithPrefix(string theStr, string prefix = "[DEFAULT]")
{
    Console.WriteLine($"{prefix} {theStr}");
}

// Aanroepen zonder prefix → gebruikt "[DEFAULT]"
PrintWithPrefix("Hello there");              // [DEFAULT] Hello there

// Aanroepen met expliciete prefix
PrintWithPrefix("Hello there", "[PREFIX]");  // [PREFIX] Hello there

Met benaamde argumenten kan er bij het aanroepen expliciet worden aangeven voor welke parameter welke waarde geldt. Dit verbetert de leesbaarheid en maakt argumentvolgorde onbelangrijk:

void PrintWithPrefix(string theStr, string prefix = "[DEFAULT]") { … }

// Gewone volgorde
PrintWithPrefix("Hi", "[!]");

// Named arguments, volgorde omwisselbaar
PrintWithPrefix(prefix: "[!]", theStr: "Hi");

Referentie en out-parameters

In C# kunnen parameters ook met behulp van een referentie of met out worden doorgegeven. Hierdoor kunnen de functies waarden in de aanroepende scope meerdere waarden kunnen teruggeven en worden aangepast.

Wanneer een parameter met ref wordt doorgegeven, ontvangt de functie geen kopie, maar een verwijzing naar de originele variabele. Wijzigingen in de functie worden daarom ook buiten de functie zichtbaar:

// Zonder ref: alleen wijziging binnen de functie
int val1 = 10;
void TestFunc(int arg)
{
    arg += 10;
}
TestFunc(val1);
Console.WriteLine(val1);  // 10

// Met ref: wijziging ook buiten de functie zichtbaar
val1 = 10;
void TestFunc2(ref int arg)
{
    arg += 10;
}
TestFunc2(ref val1);
Console.WriteLine(val1);  // 20

Een parameter met out dient uitsluitend om waarden terug te geven; de functie vult deze variabele en geeft zo meerdere waarden terug. De parametervariabele hoeft hierbij niet vóór de aanroep geïnitialiseerd te zijn.

// out-parameters voor meerdere resultaten
void PlusTimes(int x, int y, out int sum, out int product)
{
    sum = x + y;
    product = x * y;
}

int a, b;
PlusTimes(20, 20, out a, out b);
Console.WriteLine($"Sum = {a}, Product = {b}");  // Sum = 40, Product = 400

Meerdere waarden teruggeven

Een functie kan standaard slechts één waarde retourneren. De mogelijkheid bestaat om met out-parameters toch meerdere waarden terug te geven, maar dit is minder leesbaar en doorbreekt de conventie dat parameters alleen input zijn. Sinds C# 7 zijn er daarom tuples geïntroduceerd: een datastructuur om meerdere waarden onder één type terug te geven, met benoemde of anonieme elementen:

// Benoemde elementen
(int a, int b) tup1 = (5, 10);
tup1.b = 20;  // wijziging mogelijk via naam

// Anonieme elementen → Item1, Item2, …
var tup2 = ("Hallo", 3.14);
tup2.Item1 = "Nieuwe tekst";

Tuples maken het mogelijk om op een nette manier functies te definiëren die meerdere waarden teruggeven:

// Retourneert som en product als tuple
(int sum, int product) PlusTimes(int x, int y)
{
    return (x + y, x * y);
}

// Aanroepen
var result = PlusTimes(6, 12);
Console.WriteLine($"Sum = {result.sum}, Product = {result.product}");
// Output: Sum = 18, Product = 72

C# klassen definiëren

Een klasse vormt de blauwdruk voor objecten en wordt gedefinieerd met het sleutelwoord class, gevolgd door de klassennaam en een optionele access modifier (public, internal, etc.) om zichtbaarheid te regelen:

Sleutelwoord Uitleg Voorbeeld
Class Declareert een klasse 
public class Job{}

Binnen de klasse kunnen velden worden gedeclareerd om data op te slaan. In het onderstaande voorbeeld beginnen de veldnamen met een underscore om interne variabelen te onderscheiden:

public class Book
{
    private string _name;       // titel van het boek
    private string _author;     // auteur
    private int    _pageCount;  // aantal pagina’s
    // …
}

De constructor is een speciale methode die bij het aanmaken van een object wordt aangeroepen. De parameters van de constructor worden gebruikt om de interne velden te initialiseren:

public class Book
{
    // velden …

    public Book(string name, string author, int pages)
    {
        _name      = name;
        _author    = author;
        _pageCount = pages;
    }

    // …
}

Klassen bevatten ook methoden om gedrag te definiëren. In dit voorbeeld geeft GetDescription() een geformatteerde omschrijving terug via string-interpolatie:

public class Book
{
    // velden en constructor …

    public string GetDescription()
    {
        return $"{_name} by {_author}";
    }
}

In de Main-methode worden objecten van de klasse aangemaakt met het sleutelwoord new. Vervolgens kunnen de methoden worden aangeroepen:

var b1 = new Book("War and Peace", "Leo Tolstoy", 800);
var b2 = new Book("The Grapes of Wrath", "John Steinbeck", 464);

Console.WriteLine(b1.GetDescription());
Console.WriteLine(b2.GetDescription());

Acces modifiers

Bescherming van velden en methoden wordt geregeld met acces modifiers. Access modifiers bepalen op welke manier de velden, methoden en eigenschappen van een klasse toegankelijk zijn vanuit andere delen van de applicatie:

Modifier Uitleg Voorbeeld
public Toegankelijk vanuit élk codeblok in het project. 
public  void A1() { /* … */ }
protected Toegankelijk binnen de eigen klasse én afgeleide (sub)klassen. 
protected void A2() { /* … */ }
private Alleen toegankelijk binnen de eigen klasse. 
private void A3() { /* … */ }

Wanneer een private veld buiten de klasse wordt aangepast, ontstaat er een compile-error, wat aangeeft dat de access modifier verhindert dat externe code de klasse wijzigt.

Door een veld public te markeren, kan de hoofdcode het directe veld aanpassen, wat tot ongewenst gedrag kan leiden. In plaats van velden publiek te maken, kunnen getter- en setter-methoden worden toegevoegd:

public string GetName() { return _name; }
public void   SetName(string s) { _name = s; }

Hiermee blijft de interne implementatie afgeschermd en bij wijziging van de velden hoeven aanroepers niets aan te passen.

Properties definiëren

Properties combineren de rol van velden en methoden om gecontroleerde toegang tot interne data mogelijk te maken. Een property wordt gedeclareerd als een public met een datatype, klasse naam én de getter/ setter-methoden:

public class Book
{
    private string _name;

    public string Name     // property
    {
        get { return _name; }
        set { _name = value; }
    }
}

Voor eenvoudige getters en setters kan de syntax worden ingekort met de =>-notatie:

public class Book
{
    private string _author;

    public string Author
        { get => _author; set => _author = value; }
}

Properties minimaliseren de tijd die kwijtraakt aan het schrijven van aparte Get- en Set-methoden en verbeteren de leesbaarheid. Wanneer set óf get wordt weggelaten, ontstaat een read-only of write-only property en zonder backing velden, slaan auto-properties de data intern op, wat handig is voor eenvoudige data-containers. Ten slot kunnen properties ook berekende waardes teruggeven, gebaseerd op andere properties of velden:

public class Book
{
    public string Description   // alleen getter
        => $"{Name} by {Author}, {PageCount} pages";
}

Overerven

Met overerving kunnen klassen hiërarchieën vormen waarbij een bovenliggende klasse gemeenschappelijke functionaliteit bevat en een onderliggende klasse deze verder uitbreidt en specialiseert:

public class Publication
{
    private string _name;
    public string Name
    {
        get => _name;
        set
        {
            if (string.IsNullOrEmpty(value))
                throw new ArgumentException("Name cannot be blank");
            _name = value;
        }
    }
    public int    PageCount { get; set; }
    public decimal Price     { get; set; }

    public virtual string GetDescription()
        => $"{Name}, {PageCount} pages";
}

Sting representatie

Elke klasse erft impliciet van de basisklasse Object, waarin de methode ToString() wordt gedefinieerd en geeft klassen standaard een volledige klasse naam inclusief naamruimte. Standaard overschrijven ingebouwde datatypen ToString() zodat de waarde zelf wordt geretourneerd en eigen klassen zonder override tonen de namespace en klassenaam:

int x = 1000;
Console.WriteLine(x.ToString());   // "1000"

object a = new object();
Console.WriteLine(a.ToString());   // "System.Object"

Door in een eigen klasse public override string ToString() te definiëren, kan er een betekenisvolle weergave van objecten worden gecreëerd, zoals:

public class Book
{
    public string Name      { get; set; }
    public string Author    { get; set; }
    public int    PageCount { get; set; }

    public override string ToString()
        => $"Book: {Name} by {Author}";
}

Door een sting te overloaden is het mogelijk om extra parameters te accepteren voor verschillende formaten. Dit maakt flexibele weergave opties mogelijk zonder de oorspronkelijke methode te verstoren. Het overloaden van ToString() is een andere methode met een apart signatuur en is niet hetzelfde als overschrijven:

public class Book
{
    // … bestaande properties en override …

    // Condensed form (B)
    public string ToString(char format)
    {
        return format == 'B'
            ? $"{Name}: {Author}"
            : format == 'F'
                ? $"{Name} by {Author} is {PageCount} pages"
                : base.ToString();
    }
}
