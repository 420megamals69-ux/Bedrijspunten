Learning Java 11

Van start gaan met Java

Wat is Java?

Java is een programmeertaal die wordt gebruikt om instructies aan een computer te geven. Net als natuurlijke talen beschikt Java over een vocabulaire en grammaticale regels, waarmee communicatie tussen ontwikkelaar en computer mogelijk wordt gemaakt.

Met Java kunnen verschillende soorten applicaties worden ontwikkeld, waaronder webapplicaties, mobiele applicaties, desktopapplicaties, webservers en embedded systems.

Om Java-programma's uit te voeren en te ontwikkelen, is er een Java Runtime Environment (JRE) en Java Development Kit (JDK) nodig. De JRE verwijst naar een Java-omgeving voor het uitvoeren van Java-programma's en de JDK naar het volledig software-ontwikkelpakket voor het bouwen van Java-programma's.

Voor het uitvoeren van Java-programma's volstaat installatie van de JRE. Voor het ontwikkelen van Java-programma's is installatie van de JDK vereist. Bij installatie van de JDK wordt de JRE automatisch mee geïnstalleerd.

Java op een computer downloaden

Voor de fundamenten van Java is elke versie Java 8 of hoger geschikt. In de cursus wordt Java 13.0.2 gebruikt, omdat dit de nieuwste release was op het moment van opname.

Navigeer naar de Oracle-downloadpagina voor Java SE Development Kit, selecteer de gewenste versie (bijv. 13.0.2) en klik op Download. Accepteer de licentieovereenkomst en download het installatiebestand dat past bij het besturingssysteem. Voer ten slot het gedownloade installatiebestand uit en volg de installatiewizard.

De succesvolle voltooiing van de installatie kan worden gecontroleerd door in een terminal/command prompt java -version en javac -version uit te voeren.

Javac wordt gebruikt om .java-bestanden te compileren en java wordt gebruikt om gegenereerde .class-bestanden of jar-bestanden te starten.

De command line verkennen

De command line (of terminal) is een tekstgebaseerde gebruikersinterface waarmee opdrachten kunnen worden ingevoerd om de computer aan te sturen, in tegenstelling tot de grafische gebruikersinterface (GUI) waarbij met muis en vensters wordt gewerkt.

Op de Mac kan het terminal worden geopend door Spotlight Search te openen, Terminal te typen en op Enter te drukken. Op Windows kan in het Startmenu cmd worden getypt, waarna Command Prompt kan worden aangeklikt. Eenmaal in de command line kunnen de volgende basiscommando's worden gebruikt:

Je eerste Javaprogramma

Om het eerste Javaprogramma te schrijven is een teksteditor nodig (bijv. Sublime Text) om code te schrijven en in combinatie met de command line om de code te compileren en uit te voeren.

Java is een relatief verbose taal: elk programma moet een main-methode als entry point bevatten om het programma te kunnen laten functioneren:

public class HelloWorld {
    public static void main(String[] args) {
        // hier komt de uitvoercode
    }
}

De main methode kan worden uitgebreid met additionele code:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello World! It's me Vincent!");
    }
}

Nadat de geschreven code is opgeslagen, kan er in de terminal naar de map worden genavigeerd. Het compileren en uitvoeren van de code gaat als volgt:

javac Bestandnaam.java
java HelloWorld

Om Hello World vanuit de command line te laten werken moet er eerst naar de directory worden genavigeerd. Dit kan worden gedaan door de inhoud van de huidige map te tonen en naar de map te navigeren waarin HelloWorld.java staat:

Om de code te compileren wordt de javac-commando gebruikt, zodat de .java-broncode om kan worden gezet naar bytecode (.class):

java HelloWorld

Om het herhaaldelijk compileren en uitvoeren van code makkelijker te maken bestaan er tools en geïntegreerde ontwikkelomgevingen (IDE's) die deze stappen automatiseren en zo de ontwikkelaarstoolchain vereenvoudigen.

De geïntegreerde development omgeving (IDE) verkennen

Een IDE (Integrated Development Environment) is een softwarepakket dat het schrijven, compileren en uitvoeren van code ondersteunt via een grafische gebruikersinterface. Voor Java biedt een IDE geïntegreerde tools, zoals een editor, compiler en debugger, in één omgeving.

Het gebruik van een IDE voor Java heeft voordelen ten opzichte van de command line: een IDE bevat functionaliteit voor het opsporen van fouten, biedt projectstructuur en automatisch beheer van afhankelijkheden en maakt het mogelijk om code in één omgeving te bouwen en te runnen.

De IDE die in deze cursus wordt gebruikt is IntelliJ IDEA (JetBrains). De belangrijkste UI-componenten van deze IDE zijn de project-navigator, het editor venster en de run/debug-knoppen. De project-navigator toont mappen (.idea, src) en gekoppelde JDK en in het editor-venster staan de Java-bestanden klaar voor verdere bewerkingen. De run/debug-knoppen staan rechtsboven en assisteren in het testen en debuggen van de geschreven code.

Hello World in een IDE

De eerste stap om een Hello World programma in een IDE aan te maken en uit te voeren, is het aanmaken van een nieuw Java-bestand. Dit kan door in de Project-navigator links op de src map te klikken en door vervolgens Java Class te selecteren in new.

Zodra het bestand is aangemaakt moeten de standaard main methode en de Hello World uitvoerregel worden toegevoegd:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello World from the IntelliJ IDE!");    
    }
}

Ten slotte kan het programma direct in de IDE worden uitgevoerd door rechtsboven op de run knop te drukken.

Een sterke fundering bouwen

Primitieve datatypes

Data is informatie die door een computer wordt opgeslagen of verwerkt. Java classificeert deze gegevens aan de hand van datatypes, waarbij elk datatype een verzameling mogelijke waarden representeert. Deze datatypes zijn onderverdeeld in Primitieve datatypes en Reference-types.

Primitieve datatypes zijn de meest basale bouwstenen, waaronder Boolean, int, double en char. Primitive datatypes vormen de fundering voor alle overige datatypes in Java en worden hieronder nader toegelicht:

Datatype Uitleg Voorbeeld
Boolean Geeft één van twee waarden weer: true of false. 
boolean lightOn = true;
int Staat voor "integer"; geeft gehele getallen weer. 
int siblings = 3;
double Vertegenwoordigt decimale (zwevende-komma) getallen. 
double gpa = 3.4;
char Staat voor "character"; geeft één enkel teken weer. 
char initial = 'J';

Deze primitieve datatypes worden gebruikt om basiswaarden te modelleren, zoals aan/uit-statussen, aantallen, kommagetallen of losse tekens. Door deze fundamentele bouwstenen te combineren, ontstaan de complexere datastructuren en objecten in Java.

Datatypes en variabelen in Java

In Java worden losse waarden opgeslagen in variabelen, waarbij elke variabele een datatype, een naam (label) en een waarde heeft. De datatype bepaalt welke waarden zijn toegestaan, en de naam maakt het mogelijk om die gegevens later te hergebruiken. Elke variabele wordt volgens de volgende structuur gedefinieerd:

<datatype> <naam> = <waarde>;

Dit kan uiteindelijk de volgende vorm aannemen:

int studentAge = 15;
double studentGPA = 3.45;
char studentFirstInitial = 'K';
char studentLastInitial = 'H';
boolean hasPerfectAttendance = true;

Om een variabele te gebruiken, kan de waarde geprint worden met:

System.out.println(<variabeleNaam>);

Strings in Java

Naast de primitieve datatypes bestaan er reference-types, waarvan String het eerste voorbeeld is. Een String is een geordende reeks tekens en wordt gebruikt om tekst van willekeurige lengte op te slaan, bijvoorbeeld woorden of volledige zinnen:

Datatype Uitleg Voorbeeld
String Reeks geordende karakters (reference-type) 
String studentName = "Kayla Hammon";

In tegenstelling tot char (één teken) kan een String meerdere tekens bevatten.

Indexen gebruiken in Java

Binnen een String is elke positie van een teken vastgelegd met een nul-gebaseerde index. Het eerste teken staat op index 0, het tweede op index 1, enzovoort. Voor de naam "Vincent" geldt bijvoorbeeld dat V een index van 0 heeft en e een index van 4.

Om een specifiek teken uit een String te halen, biedt Java de methode charAt(). Deze methode retourneert het karakter op de opgegeven index.

Methode Uitleg Voorbeeld
charAt() retourneert het karakter op de opgegeven index. 
char studentFirstInitial = studentFirstName.charAt(0);

Met deze methode kan bijvoorbeeld eenvoudig afgeleid worden hoe uit een langere String andere gegevens, zoals initialen worden gehaald.

Strings samenvoegen in Java

Om meerdere strings en string-variabelen samen in één regel af te drukken, gebruikt Java de +-operator. Deze operator voegt links en rechts van zichzelf telkens twee tekstwaarden (zowel letterlijke strings als string-variabelen) aaneen. Dit kan er als volgt uitzien:

System.out.println(
        studentFirstName     
    + " "                  
    + studentLastName      
    + " has a GPA of "     
    + studentGPA           
);

Input en output in Java

De Scanner-klasse maakt het mogelijk voor gebruikers om gegevens in te voeren, zodat gebruikers de variabelen tijdens uitvoering kunnen bijwerken:

Klasse Uitleg Voorbeeld
Scanner Creëert een scanner. 
Scanner input = new Scanner(System.in);

Om hier gebruik van te kunnen maken moet de scanner eerst geïmporteerd worden:

import java.util.Scanner;

Daarna is het mogelijk om scanner-objecten aan te maken zoals:

System.out.println("Wat wil je instellen als nieuwe GPA?");

Scanner input = new Scanner(System.in);
studentGPA = input.nextDouble();

System.out.println(studentFirstName + " " + studentLastName 
    + " heeft nu een GPA van " + studentGPA);

Bij deze opzet moet het programma opnieuw worden uitgevoerd en handmatig aangepast voor elk item. Daarnaast blijven ingevoerde waarden slechts geldig tijdens de huidige uitvoering; waarden worden niet opgeslagen tussen sessies.

Control flow

Een programma uitplannen in de control flow

De control flow van een programma bepaalt in welke volgorde de instructies worden uitgevoerd. Tot nu toe verliep de uitvoering altijd sequentieel: elke regel code volgt direct op de voorgaande. Met speciale control flow-verklaringen kan deze volgorde worden aangepast, waardoor bepaalde regels nooit, eénmaal of meerdere malen worden uitgevoerd.

De uitvoering wordt gestuurd door voorwaarden, die bepalen óf en hoe vaak een codeblok wordt doorlopen. Dit kan bijvoorbeeld aan de hand van een if-statement:

System.out.print("Kies een getal tussen 1 en 10: ");
int inputtedNum = scanner.nextInt();

if (inputtedNum < 5) {
    System.out.println("Jouw fortune: Je hebt een rustige dag voor de boeg.");
} else {
    System.out.println("Jouw fortune: Spannende gebeurtenissen komen eraan!");
}

Operators in Java

In een beslissingsblok zoals een if-statement bestaat de voorwaarde uit een Boolean-expressie, die met behulp van relationele operatoren twee waarden vergelijkt en true of false oplevert:

Scanner scanner = new Scanner(System.in);
System.out.print("Kies een getal tussen 1 en 10: ");
int inputtedNum = scanner.nextInt();

if (inputtedNum < 5) {
    System.out.println("Jouw fortune: Je hebt een rustige dag voor de boeg.");
} else if (inputtedNum == 5) {
    System.out.println("Jouw fortune: Vandaag brengt een bijzondere verrassing.");
} else {
    System.out.println("Jouw fortune: Spannende gebeurtenissen komen eraan!");
}

Deze expressie stuurt vervolgens de control flow van het programma. Hieronder staat een overzicht van deze relationele operatoren weergeven:

Operator Betekenis Voorbeeld
< Kleiner dan inputtedNum < 5
<= Kleiner dan of gelijk aan inputtedNum <= 5
> Groter dan inputtedNum > 8
>= Groter dan of gelijk aan inputtedNum >= 8
== Gelijk aan inputtedNum == 8
!= Niet gelijk aan inputtedNum != 3

Besluitvorming met if in Java

System.out.print("Kies een getal tussen 1 en 10: ");
int inputtedNum = scanner.nextInt();

if (inputtedNum < 5) {
    System.out.println("Geniet van het geluk dat een vriend je brengt.");
} else {
    System.out.println("Je schoenkeuze zal je vandaag veel geluk brengen.");
}

De bovenstaande code bestaat uit drie stappen: het invoeren van gegevens, het controleren van de ingevoerde waarde ten opzichte van de voorwaarde en het geven van een output op basis van de voorwaarde. De besluitvorming in de code wordt afgehandeld aan de hand van een if-else statement.

Een if-statement is een control flow-structuur die code uitvoert als een Boolean-expressie true oplevert. Met else kan een alternatief codeblok worden opgegeven voor het geval de expressie false is. In één programma-executie wordt altijd óf het if- óf het else-blok uitgevoerd, nooit beide.

Scope begrijpen in Java

De scope bepaalt in welk gedeelte van een programma een variabele of code-element toegankelijk is. In Java creëert elk paar accolades ({ ... }) een nieuw blok met een eigen scope:

if (inputtedNum < 5) {
    int favoriteNumber = 5;            
} else {
}

In het bovenstaande voorbeeld is favoriteNumber alleen beschikbaar binnen de accolades van het if-blok waarin het is gedeclareerd. Buiten dit blok, zoals in het else-blok, bestaat de variabele niet.

String favoriteFood = "pizza";

if (inputtedNum < 5) {
    favoriteFood = "pasta";  
} else {
    favoriteFood = "sushi";
}

Een variabele is alleen toegankelijk binnen de scope waarin het is gedeclareerd: hoe hoger de declaratie, hoe groter het bereik waarin de variabele gebruikt kan worden. Omdat favoriteFood in het tweede voorbeeld in de parent-scope is gedeclareerd (de main-methode) is het beschikbaar binnen alle geneste blokken.

While loops in Java

Een while-lus voert herhaaldelijk een blok statements uit zolang een opgegeven Boolean-voorwaarde true blijft; bij false stopt de herhaling.

Sleutelwoord Uitleg Voorbeeld
while Definieert een while loop 
while (isOnRepeat) {
    System.out.println("Huidig nummer speelt op repeat.");
    System.out.print("Wilt u herhaling uitzetten? (ja/nee): ");

Oneindige loops kunnen ontstaan als de voorwaarde nooit false wordt. Daarom is het raadzaam om altijd een exit-mogelijkheid aan te bieden. Het gebruik van een if-statement binnen de while-lus maakt het mogelijk om de lus dynamisch te beëindigen:

import java.util.Scanner;

public class MusicPlayer {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        boolean isOnRepeat = true;  // Voorwaarde voor de while-lus

        // While-lus: herhaal zolang isOnRepeat true is
        while (isOnRepeat) {
            System.out.println("Huidig nummer speelt op repeat.");
            System.out.print("Wilt u herhaling uitzetten? (ja/nee): ");
            
            String userInput = input.nextLine();
            // Controleer of de gebruiker "ja" invoert
            if (userInput.equals("ja")) {
                isOnRepeat = false;  // Verbreek de lus bij "ja"
            }
        }
        
        // Wordt uitgevoerd nadat isOnRepeat false is geworden
        System.out.println("Speel volgend nummer.");
        input.close();
    }
}

Debuggen in Java

Syntax en logische errors debuggen

Tijdens het programmeren kunnen syntaxfouten en logische fouten plaatsvinden. Syntaxfouten ontstaan doordat de code niet kan worden gecompileerd door fouten in de structuur of "grammatica" van de code. Bij logische fouten draait de code, maar geeft het onjuiste resultaten doordat de logica van het programma fout is opgebouwd. Debuggen verwijst naar het proces van het vinden en oplossen van deze fouten.

Een strategie die gebruikt wordt voor debuggen, is het gebruik van printstatements. Hierbij worden variabelen in de console getoond om te verifiëren wat de waardes zijn tijdens de uitvoering. Syntaxfouten zijn vaak voorzien van duidelijke foutbeschrijvingen en regelverwijzingen. Het doorlezen van deze foutmeldingen helpt om potentiele syntaxfouten te debuggen.

Ook helpt het om te controleren of de toewijzingen, condities en vergelijkingen logisch correct zijn en om verschillende onverwachte en onjuist invoerwaarden op te leveren om het gedrag te observeren.

Debuggen met een IDE

Printstatements gebruiken om fouten op te sporen is effectief, maar tegenwoordig omslachtig. Moderne IDE's zoals IntelliJ IDEA, Eclipse of VS Code bieden ingebouwde debugging tools die het opsporen van fouten eenvoudiger, sneller en overzichtelijker maken.

De IDE biedt bijvoorbeeld syntaxfoutmarkeringen in realtime. Daarnaast is het mogelijk om tijdelijk de uitvoering van code op een specifieke regel te stoppen en om variabelen live te inspecteren. Dit maakt het makkelijker om het programma regel voor regel te doorlopen om het gedrag te volgen.

Functies in Java

Wat zijn functies (conceptueel)

Een functie is een benaamde verzameling van eindige stappen die samen een specifieke taak uitvoeren. Een functie heeft een reeks gedefinieerde handelingen (stappen) die nodig zijn om een taak uit te voeren en de naam waaronder deze reeks stappen bekend is. Door deze functienaam gevolgd door haakjes (()) uit te voeren, worden alle onderliggende stappen doorlopen, wat er bijvoorbeeld zo uitziet:

makePBAndJSandwich();

Nadat een functie gedefinieerd kan het overal in het programma worden hergebruikt en wijzigingen in de functiedefinitie zijn direct van toepassing op alle plekken waar de functie wordt aangeroepen.

Functies definiëren in Java

In Java moet een functie (methoden) eerst gedefinieerd worden buiten de main -methode, voordat het aangeroepen kan worden. Hierbij is het belangrijk om te beseffen dat definiëren niet hetzelfde is als aanroepen: het schrijven van de functiedefinitie bevat alleen de instructies; de functie wordt pas uitgevoerd bij een expliciete aanroep.

public static void announceDeveloperTeaTime() {
    System.out.println("waiting for developer tea time...");
    Scanner input = new Scanner(System.in);
    input.next();  // wacht op willekeurig woord + Enter
    System.out.println("It's Developer Tea Time!");
}

Zodra een methode is gedefinieerd, zoals hierboven het geval is, kan het binnen of buiten de main-methode worden uitgevoerd:

public static void main(String[] args) {
    announceDeveloperTeaTime();
}

Een functie aanroepen in Java

Nadat een methode is gedefinieerd, kan deze binnen de main-methode (of een andere methode) worden opgeroepen om de onderliggende stappen uit te voeren, zelfs als het buiten de main staat. In het onderstaande voorbeeld wordt announceDeveloperTeaTime() buiten de main gedefinieerd:

public static void main(String[] args) {
    System.out.println("Welcome to your new job");
    announceDeveloperTeaTime();   // Eerste aanroep
    System.out.println("write code, review code");
    announceDeveloperTeaTime();   // Tweede aanroep
    System.out.println("you get promoted");
}

Tijdens elke aanroep wordt de volledige methode announceDeveloperTeaTime() doorlopen, waarna de uitvoering terugkeert naar de volgende regel in main.

De main-methode is zelf ook een functie die automatisch wordt aangeroepen bij het starten van het programma. Al de overige functies worden handmatig aangeroepen. Deze functies kunnen meerdere keren hergebruikt worden in de main functie of andere functies, waardoor code‐duplicatie wordt voorkomen.

Parameters in Java

Een parameter maakt het mogelijk om bij elke aanroep andere waarden mee te geven, waardoor het resultaat van de methode afhankelijk van de input kan veranderen. Hiermee maken parameters een methode dynamisch. Elke parameter bestaat enkel binnen de methode waarin het is gedefinieerd. Het is niet toegankelijk buiten de methode en andere methoden zien deze variabelen niet.

Tijdens het definiëren van de methode komen de parameters tussen de haakjes te staan, met telkens een datatype én een naam:

public static void calculateTotalMealPrice(
    double listedMealPrice,  // parameter 1
    double tipRate,          // parameter 2
    double taxRate           // parameter 3
) {
    // … implementatie …
}

Bij het oproepen van de methode, worden de bijbehorende argumenten in dezelfde volgorde tussen de haakjes geplaatst:

calculateTotalMealPrice(15.0, 0.20, 0.08); 
calculateTotalMealPrice(25.0, 0.18, 0.08);

Return types in Java

Met returntypes kan een methode berekende gegevens terug leveren, waardoor die gegevens in andere delen van het programma verder kunnen worden benut:

Sleutelwoord Uitleg Voorbeeld
return Levert een waarde van het opgegeven returntype op. 
return result;

Waar een void-methode niets teruggeeft, retourneert een niet-void-methode een waarde van het opgegeven type.

public static double calculateTotalMealPrice(
    double listedMealPrice,
    double tipRate,
    double taxRate
) {
    double result = listedMealPrice 
                    + (listedMealPrice * tipRate) 
                    + (listedMealPrice * taxRate);
    return result;  // levert een double op
}

Alle methoden in Java hebben een returntype. Door het returntype van void naar double te wijzigen, wordt de lokale variabele result buiten de methode beschikbaar. De geretourneerde double kan tevens worden opgeslagen in een variabele van hetzelfde type:

public static void main(String[] args) {
    double groupTotalMealCost = calculateTotalMealPrice(100, 0.20, 0.08);
    System.out.println(groupTotalMealCost);  // drukt 128.0 af

    double individualTotalMealPrice = groupTotalMealCost / 5;
    System.out.println(individualTotalMealPrice);  // drukt 25.6 af
}

De meest voorkomende returntypes zijn hieronder weergeven:

Returntype Uitleg Voorbeeld
void Geeft geen waarde terug 
public static void log(String msg) { … }
int Geheel getal 
public static int sum(int a, int b) { … }
double Kommagetal 
public static double avg(double a, double b) { … }
boolean Waar of onwaar 
public static boolean isEven(int n) { … }
char Enkel teken 
public static char firstChar(String s) { … }
String Tekststring 
public static String greet(String name) { … }

Ingebouwde functies gebruiken in Java

Ingebouwde functies zijn de vooraf gedefinieerde methoden die de programmeertaal of de standaardbibliotheek aanbiedt. Ontwikkelaars hoeven deze methoden niet zelf te implementeren, maar kunnen deze direct aanroepen. De println() en equals() functies zijn voorbeelden van ingebouwde functies.

In tegenstelling tot gebruikers-gedefinieerde functies, die vaak direct op klasse niveau staan, wordt de punt-operator bij ingebouwde functies gebruikt om eerst een klasse of object en vervolgens de methode aan te wijzen:

// Printen met de ingebouwde println-functie
System.out.println("Hallo wereld!");

// Vergelijken van twee strings
boolean gelijk = "test".equals(inputString);

// Exponentiëren met Math.pow
double resultaat = Math.pow(2, 5);  // geeft 32.0

De ingebouwde functies zijn grondig getest door de makers van Java en in het geval dat een actie uitgevoerd kan worden met een ingebouwde functies is er geen eigen implementatie nodig. Hierdoor kan er meer op de applicatielogica worden gefocust in plaats van op her-implementatie van standaardfunctionaliteit.

Klassen in Java

Klassen in Java

Een klasse is een door de gebruiker gedefinieerde blauwdruk die de attributen en methoden omvat die het object representeren. Alle objecten die op basis van een klasse worden gemaakt, erven dezelfde structuur, maar kunnen verschillende waarden voor de attributen bevatten. Elke klasse-definitie bevat variabelen die de kenmerken van elk object vastleggen:

public class Triangle {
    private double base, height, sideA, sideB, sideC;

    public Triangle(double base, double height, 
                    double sideA, double sideB, double sideC) {
        this.base   = base;
        this.height = height;
        this.sideA  = sideA;
        this.sideB  = sideB;
        this.sideC  = sideC;
    }

    public double findArea() {
        return 0.5 * base * height;
    }

    public String calculateTriangleType() {
        if (sideA == sideB && sideB == sideC)      
        return "Equilateral";
        
        else if (sideA == sideB || sideB == sideC || sideA == sideC)                   
        return "Isosceles";
        
        else                                       
        return "Scalene";
    }

    public static boolean isValidTriangle(double a, double b, double c) {
        return a + b > c && a + c > b && b + c > a;
    }
}

Attributen zijn variabelen die de kenmerken van een object vastleggen, zoals lengte of kleur. Methoden beschrijven wat een object kan doen of welke berekeningen eraan verbonden zijn en zijn in feite functies binnen de klasse. Soms kan een berekende eigenschap ook als attribuut gelden. Het ontwerp bepaalt in dit geval of iets als methode of attribuut wordt geïmplementeerd, afhankelijk van hergebruik en opslagbehoefte.

Methoden die géén gebruikmaken van de instantie specifieke attributen, maar wel thematisch bij de klasse horen, krijgen het sleutelwoord static. Een methode met static kan worden aangeroepen zonder een object te hoeven creëren:

Constructors in Java

Een constructor is een methode binnen een klasse nieuwe object­instanties maakt en initialiseert. Elke klasse heeft een constructor, die expliciet kan worden gedefinieerd om de attributen direct bij creatie een waarde te geven. Met constructors blijft de code overzichtelijk en wordt elk object direct bij creatie correct ingesteld.

Bij het definiëren van een constructor wordt per parameter een datatype en naam opgegeven, waarna de corresponderende attributen worden geinitialiseerd:

public class Triangle {
    private double base, height, sideA, sideB, sideC;

    // Constructor: parameters komen overeen met attributen
    public Triangle(double base, double height,
                    double sideA, double sideB, double sideC) {
        this.base   = base;    // 'this.base' = attribuut, 'base' = parameter
        this.height = height;
        this.sideA  = sideA;
        this.sideB  = sideB;
        this.sideC  = sideC;
    }

    // Methode om oppervlakte te berekenen
    public double findArea() {
        return 0.5 * base * height;
    }
}

Een klasse creëren in Java

De eerste stap in het creëren van de klasse in deze cursus, is het aanmaken van een nieuw Java-bestand, waarin een klasse wordt gedefinieerd die dezelfde naam als het bestand heeft. Vervolgens worden de eigenschappen die de klasse heeft gedefinieerd met behulp van variabelen die later bij elke instantie een eigen waarde krijgen.

Daarna wordt elk attribuut op basis van parameters geinitialiseerd, waarbij this wordt gebruikt om onderscheid te maken tussen de parameternaam en de attribuutnaam. Ten slot wordt een methode toe gevoegd die géén parameters nodig heeft en de waarde retourneert.

public class Triangle {
    private double base;
    private double height;
    private double sideA;
    private double sideB;
    private double sideC;

    public Triangle(double base, double height,
                    double sideA, double sideB, double sideC) {
        this.base   = base;
        this.height = height;
        this.sideA  = sideA;
        this.sideB  = sideB;
        this.sideC  = sideC;
    }

    public double findArea() {
        return this.base * this.height / 2;
    }
}

Een instantie creëren in Java

De eerste stap in het creëren van een instantie is het aanmaken van een nieuw Java-bestand met daarin een main-methode. Vervolgens worden de benodigde variabelen binnen de main gedeclareerd en wordt de constructor aangeroepen met de gewenste parameterwaarden, bijvoorbeeld als volgt:

public static void main(String[] args) {
    // Eerste driehoek: base=15, height=8, sides=15, 8, 17
    Triangle triangleA = new Triangle(15, 8, 15, 8, 17);

    // Tweede driehoek: base=3, height=2.598, sides=3, 3, 3
    Triangle triangleB = new Triangle(3, 2.598, 3, 3, 3);

    // Voorbeeld: oppervlakte berekenen van triangleA
    double areaA = triangleA.findArea();
    System.out.println("Oppervlakte triangleA: " + areaA);
}

Om objecten te creëren op basis van een klasse blauwdruk, wordt het new-sleutelwoord in een apart uitvoerbaar programma gebruikt. Zo blijft de definitie van de klasse gescheiden van de code die de instantie(n) daadwerkelijk aanmaakt en gebruikt.

Instantie methodes vs. Klasse methodes

Java bevat instantie methodes en klasse methodes. Instantie methodes gebruiken de attributen van een specifieke instantie en vereisen daarom een bestaand object. Deze methodes worden aangeroepen met de punt­operator op een object­variabele, bijvoorbeeld:

Triangle triangleA = new Triangle(15, 8, 15, 8, 17);
double area = triangleA.findArea();

Klasse methodes hebben geen toegang tot instantie­attributen (this) en werken los van individuele objecten, waardoor deze methodes zonder object­instanties worden aangeroepen. Deze methodes worden aangeroepen met de punt­operator op de klasse naam, bijvoorbeeld:

double result = Math.pow(2, 3);

Instantie methodes gebruiken in Java

Instanties methodes worden opgeroepen doormiddel van een specifiek object en gebruiken de attributen van dat object voor de berekeningen. Concreet houdt dit in dat object.method() wordt gebruikt tijdens het aanroepen.

Omdat iedere instantie een eigen attributen heeft, levert een tweede aanroep per object een verschillend resultaat op. Instanties methodes kunnen worden gedebugd door breakpoints voor de aanroep, binnen de methode en na de return te plaatsen.

Instantie en klasse variabelen in Java

Java heeft variabelen die per object verschillen (niet-static) en variabelen die één waarde voor de hele klasse vastleggen (static). De instantie variabelen (niet static) horen bij een object­instantie en bevatten data die per instantie kan verschillen. Deze variabelen worden aangeroepen met behulp van de volgende structuur:

object.variabeleNaam

De klasse variabelen (static) horen bij de klasse zelf en behouden één gedeelde waarde voor alle instanties. Deze variabelen worden aangeroepen met behulp van de volgende structuur:

ClassName.variabeleNaam

Review: Klassen vs. Instanties

Een klasse bevat zowel attributen (properties) als gedrag (methods) en elke klasse heeft een constructor. Een constructor is een speciale methode om object­instanties te creëren en te initialiseren.

Een instantie is een concreet object dat ontstaat door de constructor van een klasse aan te roepen met new ClassName(). De attributen die per object verschillend zijn, zoals de hoogte of de kleur, worden Instantie variabelen genoemd. Instantie methoden gebruiken deze instantievariabelen en worden aangeroepen met object.method().

Klasse variabelen behouden één waarde voor alle instanties van een klasse (bijv. Triangle.numOfSides). De klasse methoden kunnen zonder object­instantie worden aangeroepen, met ClassName.method() en zijn ideaal voor hulpfuncties en constante waarden die niet per object mogen variëren.

Deze begrippen zullen vaak terugkomen bij het debuggen, in de documentatie en in zoekopdrachten, waardoor een goed begrip van deze concepten essentieel is in het leren en toepassen van Java.
