De rol van HTML

Een website die vandaag gecodeerd wordt, kan over vijf of tien jaar nog steeds werken zonder updates. De code kan functioneren op oude computers en browsers, en het internet draait op diverse apparaten zoals Mac, Windows, iOS, Android, Linux, e-boeklezers, gameconsoles, televies en horloges. Deze veelzijdigheid is mogelijk dankzij de drie verschillende programmeertalen die elk een specifiek deel van het werk doen: HTML, CSS en Javascript.

HTML staat voor HyperText Markup Language. HTML markeert de inhoud van een site en vertelt de computer wat deze inhoud is. HTML bevat geen programmeerlogica, loops of functies, omdat het een declaratieve taal is. Het declareert simpelweg wat een browser moet doen. Daarnaast is HTML instaat om resultaten op te leveren zelfs als iets ontbreekt of verkeerd is gespeld. HTML gokt in deze scenario's wat er wordt bedoeld, repareer en weergeeft de pagina alsnog. Van de drie talen is HTML het meest robuust.

CSS staat voor Cascading Style Sheets en verzorgt de visuele stijl en lay-out voor alles op de webpagina. Het definieert bijvoorbeeld kleuren, typografie en grootte, en maakt het mogelijk om eenvoudige animaties en interacties aan een webpagina toe te voegen. CSS bevat wat computerwetenschappelijke structuur waardoor de taal kwetsbaarder, maar krachtiger is. Als er iets misgaat in CSS, slaat de browser dat gedeelte van de code over. De rest van de stijlen worden echter wel toegepast. Het resultaat is misschien niet helemaal correct, maar het grootste gedeelte zal werken.

Javascript is een programmeertaal die het mogelijk maakt om krachtigere interactiviteit te creÃ«ren. Hoe complexer en rijker een interface wordt, hoe waarschijnlijker het is dat Javascript is gebruikt. Van de drie talen is Javascript het krachtigst en het kwetsbaarst. Als er iets mis is met de code, of als de browser te oud is om te weten wat die code betekent, zal de browser de Javascript code niet uitvoeren.

Ontwikkelaars kunnen de kwaliteiten van deze talen combineren. Als iets met HTML kan worden gedaan, wordt het ook met HTML gedaan aangezien dit het robuustere deel van de stack is. De rest van de talen worden gebruikt, als HTML ergens niet toe instaat is.

De syntax van HTML-elementen

Inhoud in HTML wordt gedefinieerd doormiddel van een HTML-tag. Een tag begint met een kleiner-dan haak gevolgd door een letter, woord of afgekort woord, en eindigt met een groter-dan haak. Een HTML-tag voor een paragraaf ziet er als volgt uit: <p>

Er zijn twee soorten tags: opening tags en closing tags. De bovenstaande tag is een voorbeeld van een opening tag. De closing tag lijkt hierop, maar bevat een extra schuine streep, dus: </p>. De opening tag markeert het begin, de inhoud staat in het midden en de closing tag markeert het einde. Denk bijvoorbeeld aan: <p>Dit is een paragraaf.</p>

De opening en closing tags vormen samen een element. Alhoewel niet elk element een closing tag heeft, hebben veel HTML-elementen zowel een opening als een closing tag. Het vergeten van de closing tag aan het einde van een element is een van de meest gemaakte fouten.

Het is mogelijk om een HTML-element in het ander HTML-element te nesten, bijvoorbeeld: <p>Deze paragraaf bevat <em>benadrukte tekst</em> voor het effect.</p>. Een HTML-document bestaat in feite volledig uit elementen die in elkaar zijn genest. Hier is nog een praktisch voorbeeld van hoe HTML is genest:

<article>
    <h1>Dit is een headline</h1>
    <p>Dit is de eerste paragraaf.</p>
    <p>Dit is de tweede paragraaf met <em>benadrukte tekst</em> voor het effect.</p>
</article>

Alles wat na de closing tag van het artikel komt, wordt weergegeven op dezelfde HTML-pagina. Door de closing tag begrijpt de browser echter dat de rest buiten dit artikel valt. De browser houdt namelijk rekening met de structuur van hoe HTML-elementen genest zijn en creÃ«ert een uitgebreide stamboom van wat met wat verwant is en hoe: Dit is de DOM.

Headlines

Headlines helpen om de inhoud van de pagina op te splitsen, zodat mensen gemakkelijker kunnen begrijpen wat de structuur en de informatie op een pagina is. HTML bevat 6 elementen voor het identificeren van headlines: h1, h2, h3, h4, h5 en h6. Elk van deze headlines verschild visueel van de ander. H1 is de grootst en het meest prominente, h6 is de kleinste en het meest subtiel en de rest valt daar tussenin.

Element Gebruikscontext Voorbeeld

Headlines voor een tekst worden niet gekozen op basis van hoe het eruit ziet, aangezien het aanpassen van stijlen mogelijk is met CSS. In plaats daarvan wordt een headline gekozen op basis van wat het betekent. Denk bijvoorbeeld aan h1 hebben voor een titel en h2 voor de subtitels. Intentioneel zijn met het gebruik van headlines heeft bijvoorbeeld een grote invloed op gebruikers die met een schermlezer de pagina navigeren.

Een attribuut kan aan een element worden toegevoegd om het meer functionaliteit te geven. Sommige attributen zijn alleen toepasbaar op een bepaald elementen, zoals het datetime attribuut. De weergave van HTML wordt meestal niet beÃ¯nvloed door spaties, tabs en regeleinden. Er zijn een paar uitzonderingen zoals de pre-, code-, en textarea-elementen of CSS, maar meestal negeert de browser echter allesbehalve enkele spaties.

Naarmate HTML zich ontwikkelde, groeide de hoeveelheid geheugenruimte in computers. Code gemakkelijk leesbaar en begrijpelijk maken, werd daarom belangrijker dan het besparen van geheugenruimte.

De nieuwere HTML-elementen hebben altijd een opening- en closingstag. Sommige oudere elementen bevatten echter geen closingstag. Ontwikkelaars kunnen ervoor kiezen om een afsluitende schuine streep in deze elementen te gebruiken of niet. Browsers ondersteunen beiden opties.

URL-paden

Een relatieve URL is bruikbaar wanneer hetgeen waarnaar wordt gelinkt op dezelfde site en in hetzelfde domein is. Het gebruik van relatieve URL's is nuttig voor een project dat van server naar server beweegt terwijl eraan wordt gewerkt. De domeinnaam wordt weggelaten in een relatieve URL, wat er zo uit kan zien: /user/profile.html.

Het bovenstaande voorbeeld is een relatieve URL die leidt naar een bestand genaamd profile.html in een folder genaamd user. In tegenstelling tot bestanden hebben folders geen extensie zoals .html, .jpg, .css, of .js waardoor "user" genoeg is in het bovenstaande voorbeeld. Met de schuine streep aan het begin wordt aan geduid dat de browser naar het hoogste niveau van de bestandsstructuur moet gaan.

De andere schuine strepen in de URL geven aan dat de browser een niveau naar beneden moet gaan in de bestandstructuur. Het is echter ook mogelijk om een URL-pad te maken dat relatief is ten opzichte van het bestand waarin de link is geschreven. Hiervoor wordt ../ aan het begin gebruikt. Dit geeft aan dat er een niveau omhoog moet worden gegaan.

De browser zoekt en laadt automatisch een index.html-bestand wanneer het de instructies krijgt om een URL-folder in te gaan, ongeacht of de URL relatief of absoluut is. Het is mogelijk om nette URL's te maken, door bijvoorbeeld een bestand genaamd index.html in een folder met de naam profile index.html te plaatsen. Hierdoor hoeft een bestand niet profile.html te worden genoemd. Dit werkt echter alleen voor HTML en bijvoorbeeld niet voor afbeeldingen of CSS.

Ten slotte maakt het niet uit of de laatste schuine streep wel of niet in een URL wordt opgenomen. Dit betekent dat deze drie URL's allemaal naar dezelfde pagina zullen leiden:

<a href="/folder/index.html">Folder</a>
<a href="/folder/">Folder</a>
<a href="/folder">Folder</a>

Srcset kan op twee manieren afbeeldingen keuzes maken: Op basis van de pixeldichtheid of op basis van pixeldichtheid en viewport-breedte. De browser maakt keuzes op basis van de pixeldichtheid door de gewenste resolutie te specificeren met bijvoorbeeld 2x of drie 3x.

De browser kan een keuze maken op basis van de breedte van de viewport en pixeldichtheid door de breedte van elk bestand te specificeren. Denk bijvoorbeeld aan 480w of 960w, wat staat voor 480 en 960 pixels breed:

In het bovenstaande voorbeeld wordt de afbeelding echter niet gekozen op basis van de pagina lay-out. De browser maakt namelijk de veronderstelling dat de breedte van de afbeelding 100% van de viewport moet zijn. Het afbeeldingsbestand wordt namelijk voor de CSS geladen en voordat de lay-out is opgemaakt. De browser weet tijdens het downloaden dus niet hoe groot het object is waarin de afbeelding wordt geplaatst.

Tabel data structureren

Toepassen van tabellen

Een tabel illustreert het onderlinge verband tussen verschillende data-cellen. Het tabel-element wordt alleen ingezet voor deze semantische reden. Het tabel-element wordt niet gebruikt met het doel om de stijl van een stuk tekst aan te passen, aangezien dit voor toegankelijkheidsproblemen zorgt. Daarom wordt in deze gevallen CSS gebruikt.

CSS kan wel worden gebruikt om de manier waarop een tabel wordt weergegeven te herschikken, zodat het visueel anders oogt dan een conventionele tabel. Dit is vooral relevant in de context van kleinere schermen.

Tabel rijen bouwen

Een HTML-tabel bestaat uit de volgende HTML-elementen: tabel, tr, th en td.

Element Gebruikscontext Voorbeeld

<table>
    <tr>
        <th>Vogel</th>
        <th>Kleur</th>
        <th>Photo</th>
    </tr>
    <tr>
        <td>Meeuw</td>
    </tr>
    <tr>
        <td>Raaf</td>
    </tr>
</table>

<tr>
    <td>Meeuw</td>
    <td>Wit</td>
</tr>
<tr>
    <td>Raaf</td>
    <td>Zwart</td>
</tr>

<tr>
    <th>Vogel</th>

</tr>

Om de stijl van een tabel aan te passen wordt CSS toegepast. CSS wordt bijvoorbeeld gebruikt om een header volledig in hoofdletters te weergeven. Dit zorgt ervoor dat de woorden correct worden uitgesproken.

Aanvullende attributen kunnen worden gebruikt om een geavanceerde HTML-tabel te maken. Dit maakt het bijvoorbeeld mogelijk om de inhoud van een tabel te spreiden over meerdere rijen of kolommen. Daarnaast kan een header, body en footer van een tabel worden gedefinieerd en kan er een bijschrift worden toegevoegd.
